# PREDICTFUN_OPERATIONS.md

Операционный паспорт модуля PredictFun.

## Назначение
PredictFun = рабочий рыночный/ордерный модуль.

Не смешивать с `News Monitor`.

## Текущие рабочие слои

### 1. Private summary (active)
- Timer: `openclaw-predictfun-private-summary.timer`
- Частота: hourly
- Назначение: полная сводка по рынкам, текущим ордерам и действиям
- Формат: см. `docs/OUTPUT_FORMATS.md` / PredictFun hourly summary

### 2. Trigger watcher (active)
- Timer: `openclaw-predictfun-trigger.timer`
- Частота: каждые 10 минут
- Назначение: только срочные сигналы на перестановку
- Формат: см. `docs/OUTPUT_FORMATS.md` / PredictFun 10-minute trigger
- Если сигналов нет — молчит
- Если временно недоступен RPC/auth — цикл пропускается без падения сервиса

### 3. Proposal / confirm layer (prepared)
- Script: `scripts/predictfun_replace_plan.py`
- Назначение: собрать pending plan на перестановку и подготовить сообщение подтверждения
- Статус: каркас готов

### 4. Execute pending plan (prepared)
- Script: `scripts/predictfun_execute_pending.py`
- Назначение: отменить старые ордера и выставить новые по pending plan
- Статус: каркас готов, execution включать только по подтверждению пользователя

## Рабочая логика перестановки
- Бюджет на сторону: `$118`
- Целевая цена:
  - Yes = текущий рынок Yes - 3¢
  - No = текущий рынок No - 3¢
- Trigger на перестановку:
  - если по уже выставленному ордеру отклонение до новой целевой цены >= 2¢
- Если по стороне ордера нет:
  - это не ошибка
  - по умолчанию = `✅ без изменений`

## Режим SELL_HOLD_MODE
- Если buy-ордер исполнен, модуль переходит в `SELL_HOLD_MODE`.
- Для пользовательского языка событие fill фиксируется как `✅ налили` или нейтрально `покупка исполнена`; формулировку `залили` не использовать как эталон.
- После fill модуль должен один раз прислать событие в рабочем формате рынка, а не абстрактный системный текст.
- После этого модуль не должен спамить одно и то же сообщение каждый час.
- Когда пользователь выставляет sell-ордер, модуль должен найти его и прислать sell-уведомление в рабочем формате PredictFun.
- Пока sell-ордер активен, модуль тихо мониторит его без спама.
- Когда sell-ордер исполнен, модуль должен прислать событие продажи и попросить пользователя дать команду на новое выставление buy-ордеров.
- После этого возможны 2 рабочих сценария:
  1. пользователь пишет `выстави заново ордера` / `выстави сам` → модуль сам выставляет buy-ордера по закреплённым правилам;
  2. пользователь пишет `ордера выставил сам` → модуль находит новые buy-ордеры и дальше снова мониторит/предлагает перестановки с подтверждением пользователя.
- Автоматическая перестановка buy-ордеров после подтверждения пользователя должна работать через сохранённый replace-plan (`memory/predictfun-replace-plan.json`) и execute-layer, который:
  - отменяет старый buy-ордер через `POST /v1/orders/remove`;
  - выставляет новый limit buy по тем же правилам цели;
  - очищает plan после успешного выполнения;
  - не исполняет просроченный plan (`STALE_PLAN`).
- Команда `отмени все ордера на покупку` должна идти через тот же private API flow `/v1/orders/remove` со списком всех active buy order ids.
- Пока sell-ордер не исполнен и новых buy-ордеров нет, silence = нормальное поведение, а не ошибка.

## Эталонный пользовательский flow и форматы

### 1. Выставление buy-ордеров
Формат уведомления при успешной постановке:

```text
MetaMask – June 30, 2026
• рынок: Yes 12.3¢, No 87.8¢, spread 0.1¢
• твои ордера: Yes 9.3¢, No 84.8¢
❗️Что сделать:
Yes – ✅ выставлено
No – ✅ выставлено
```

Правила:
- без вводных фраз вроде `принял`, `ставлю`, `делаю`, `верну результат`;
- сразу только финальное уведомление;
- если рынков несколько, каждый рынок идёт отдельным блоком.

### 2. Обычный мониторинг
После успешного выставления или после успешного replace модуль возвращается в обычный monitoring flow.

Формат:

```text
MetaMask – June 30, 2026
• рынок: Yes 12.4¢, No 87.6¢, spread 0.3¢
• твои ордера: Yes 9.5¢, No 84.6¢
❗️Что сделать:
Yes – ✅ без изменений
No – ✅ без изменений
```

### 3. Replace-сигнал
Если рынок ушёл достаточно далеко, модуль предлагает перестановку.

Формат:

```text
MetaMask – June 30, 2026
• рынок: Yes 13.8¢, No 86.3¢, spread 0.2¢
• твои ордера: Yes 9.3¢, No 84.8¢
❗️Что сделать:
Yes – ♻️ переставить на 10.8¢
No – ✅ без изменений

Подтверждаешь переставление?
```

Правила:
- вопрос `Подтверждаешь переставление?` обязателен в конце replace-сообщения;
- после подтверждения модуль не остаётся в confirm-mode, а возвращается к обычному monitoring flow.

### 4. Replace исполнен
После подтверждения пользователя:

```text
MetaMask – June 30, 2026
• рынок: Yes 13.8¢, No 86.3¢, spread 0.2¢
• твои ордера: Yes 10.8¢, No 84.8¢
❗️Что сделать:
Yes – ✅ переставлено
No – ✅ без изменений
```

### 5. Buy fill → SELL_HOLD_MODE
Когда покупка исполнена:

```text
MetaMask – June 30, 2026
• рынок: Yes 13.8¢, No 86.3¢, spread 0.2¢
• твои ордера: Yes – исполнен, No 84.8¢
❗️Что сделать:
Yes – ✅ налили
No – ⏸️ пауза

Переходим в SELL_HOLD_MODE
```

### 6. Наблюдение за sell-ордером
Когда sell-ордер активен:

```text
MetaMask – June 30, 2026
• рынок: Yes 22.4¢, No 77.7¢, spread 0.1¢
• твои ордера: Sell Yes 29.0¢
❗️Что сделать:
Sell – 👀 наблюдаю
No – ⏸️ пауза
```

### 7. Sell исполнен
Когда продажа исполнена:

```text
MetaMask – June 30, 2026
• рынок: Yes 29.1¢, No 70.9¢, spread 0.1¢
• твои ордера: Sell Yes – исполнен
❗️Что сделать:
Sell – ✅ продалось
Напиши: выстави заново ордера
```

### 8. Массовая отмена buy-ордеров
Для команды пользователя на остановку buy-части модуль должен уметь отменять все buy-ордера и давать короткий итог без лишнего текста.

## Отключено
### PredictFun public text alert layer
- Timer: `openclaw-predictfun-public-watch.timer`
- Причина: старый текстовый слой `✔️ PredictFun Alert ...` признан неинформативным

## Правило изменений
Перед любым изменением PredictFun:
1. проверить текущие timer/service/script
2. сверить форму вывода
3. не плодить дубль контуров
4. после правки сделать контрольный прогон
5. проверить, что старый шум не вылезает снова
